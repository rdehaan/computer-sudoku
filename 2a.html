Trial and error is (often) effective

Let's start with the simplest algorithm that combines several of computers' strong suits: pattern matching, bookkeeping and brute speed.

The main idea of this algorithm is a backtracking search: just try putting some digit somewhere, and if this doesn't lead to a solution, go back and try something else. After having tried something, the algorithm also uses some basic patterns (naked and hidden singles) to draw some further conclusions.

We will dive into the working of the algorithm in more detail soon, but let's first have a look at the algorithm in action. It solves Sudoku A in only a few steps. (Remember that Sudoku A is harder than the Sudoku puzzles that you typically find in newspapers.)
